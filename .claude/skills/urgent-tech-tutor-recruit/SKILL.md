# 기술튜터 긴급구인 DM 스킬

`/urgent-tech-tutor-recruit` 실행 시, 지정된 교육 일자에 **가용 가능한 기술튜터 목록을 ax-hub에서 조회**해 사용자에게 먼저 보여주고, 사용자가 그중 **선별한 인원에게만** 개별 Slack DM으로 참여 가능 여부를 문의합니다.

기존 `recruit-share`(채널에 공개 구인 글 게시)와 달리, 이 스킬은 **특정 인원을 지목해 1:1 DM**으로 문의하는 용도입니다 (급하게 몇 명만 필요한 상황).

---

## 전체 흐름 요약

```
[1단계] 인자 파싱 (기업명 / 교육명 / 장소 / 일정 / 시간 / 담당자)
    ↓
[2단계] ax-hub에서 해당 일정에 가용 가능한 기술튜터 조회 → 표로 제시
    ↓
[3단계] 사용자가 DM 보낼 인원을 선별해 알려줌 (필수 대기 — 자동 선별 금지)
    ↓
[4단계] 선택 인원의 slack_id 확인 → 개별 DM 초안 미리보기
    ↓
[5단계] 사용자 확인 후 각자에게 개별 Slack DM 전송
```

---

## 1단계: 인자 파싱

명령 인자에서 아래 항목을 추출합니다. (웹앱 버튼은 이 형식으로 명령을 복사합니다.)

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 기업명 | 교육 발주 기업 | 권장 |
| 교육명 | 교육 과정명 | 필수 (DM 본문에 사용) |
| 장소 | 교육 장소 | 필수 |
| 일정 | 콤마로 구분된 날짜 목록 (`YYYY-MM-DD,YYYY-MM-DD`) | 필수 |
| 시간 | `HH:MM~HH:MM` | 권장 (없으면 시간 문구 생략) |
| 담당자 | 발신자 이름(웹앱 localStorage `training_owner` 값) | 필수 — 없으면 사용자에게 질문 |

누락된 필수 항목은 한 번에 모아서 질문합니다.

---

## 2단계: 가용 기술튜터 조회

1단계에서 얻은 날짜 목록으로 아래 SQL을 실행합니다. `ARRAY[...]` 안에는 일정의 모든 날짜를 채웁니다.

```sql
SELECT i.id, i.name, i.phone, i.slack_id,
       array_agg(a.date ORDER BY a.date) FILTER (WHERE a.status = 'available') AS available_dates
FROM instructors i
JOIN instructor_segments s ON s.instructor_id = i.id
JOIN availability a ON a.instructor_id = i.id
WHERE s.role = 'tech'
  AND i.is_active = true
  AND a.date = ANY(ARRAY['2026-07-15','2026-07-16']::date[])
  AND a.status = 'available'
GROUP BY i.id, i.name, i.phone, i.slack_id
ORDER BY array_length(array_agg(a.date) FILTER (WHERE a.status = 'available'), 1) DESC, i.name;
```

- `available_dates`의 길이가 요청한 전체 날짜 수와 같은 사람 = **전체 가능**.
- 일부만 일치하는 사람 = **일부 가능** (어느 날짜가 가능한지 함께 표시).
- 결과를 **전체 가능 → 일부 가능** 순으로 정리해 사용자에게 표로 보여줍니다:

```
{교육명} ({일정}) 기술튜터 가용 현황

전체 가능 (N명)
- {이름} ...

일부만 가능
- {이름}: {가능 날짜}만 가능
```

- 조회 결과가 없으면 그대로 안내하고 종료합니다.

---

## 3단계: 사용자 선별 대기

**절대 자동으로 인원을 선택하지 않습니다.** 표를 보여준 뒤 아래처럼 질문하고 사용자 응답을 기다립니다:

> **이 중 DM을 보낼 인원을 알려주세요.**

사용자가 이름을 알려주면(예: "조효은, 김성진, 우민정") 해당 인원만 다음 단계로 진행합니다.

---

## 4단계: DM 초안 미리보기

선택된 인원의 `slack_id`를 2단계 조회 결과에서 확인합니다.

- `slack_id`가 없는 인원은 "슬랙 ID 미등록으로 자동 발송 불가"로 안내하고 DM 대상에서 제외합니다.

각 인원에게 아래 템플릿으로 **이름만 다르게** DM을 작성합니다:

```
**[{교육명} 기술튜터 참여 가능 여부 문의]**

안녕하세요, {이름}님. 팀스파르타 {담당자} 매니저입니다.

다음 교육의 기술튜터 참여가 가능하신지 확인 차 연락드립니다.

- 교육명: {교육명}
- 일정: {일정 — "YYYY-MM-DD(요일) ~ YYYY-MM-DD(요일)" 또는 개별 날짜 나열}
- 시간: {시간} (시간 인자 없으면 이 줄 생략)
- 장소: {장소}

{일정에 포함된 날짜 모두} 참여 가능하신지 회신 부탁드립니다. 감사합니다.
```

전송 대상자 목록과 메시지 내용을 미리보기로 출력하고 아래 문구로 마무리합니다:

> **위 내용으로 {N}명에게 개별 DM을 전송할까요? 수정할 내용이 있으면 말씀해 주세요.**

---

## 5단계: 개별 Slack DM 전송

사용자가 전송을 확인하면(예: "전송해줘", "네", "검수완료"):

1. 대상자 각각에게 `slack_send_message`로 `channel_id`에 해당 인원의 `slack_id`를 지정해 개별 발송합니다. (채널이 아닌 1:1 DM — user_id를 channel_id로 사용)
2. 전송 완료 후 아래처럼 결과를 정리해 보여줍니다:

> **✅ {N}명에게 개별 DM을 전송했습니다: {이름 목록}**
> (slack_id 미등록으로 제외된 인원이 있으면 함께 안내)

전송 실패한 인원이 있으면 오류 내용과 함께 안내하고 재시도 여부를 묻습니다.

## 사용 예시

```
/urgent-tech-tutor-recruit 기업명:삼성전자로지텍 교육명:삼성전자로지텍 임원 교육 장소:경기도 수원시 영통구 삼성로 290 삼성전자로지텍 본사 일정:2026-07-15,2026-07-16 시간:09:00~15:30 담당자:송찬호
→ ax-hub 조회 → 가용 현황 표 출력 → 사용자가 "조효은, 김성진, 우민정" 선택 → DM 미리보기 → 확인 → 3명에게 개별 전송
```

---

## 주의사항

- `mcp__ax-hub__query_sql`은 SELECT만 허용됩니다 (read-only).
- 2단계 조회 결과를 사용자에게 보여주기 전까지 DM을 발송하지 않습니다.
- 인원 선별은 반드시 사용자가 직접 지정합니다 — 스킬이 임의로 "전체 가능" 인원을 자동 선택해 보내지 않습니다.
- `slack_id`가 없는 인원은 발송 대상에서 제외하고 반드시 안내합니다.
- DM은 채널이 아닌 1:1 대화로 전송합니다 (`channel_id`에 대상자의 Slack user_id 사용).
- 사용자가 전송을 확인하기 전까지 미리보기 직후 자동 전송 금지.
