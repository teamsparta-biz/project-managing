# 세금계산서 발행 확인 메일 초안 생성 스킬

사용자가 `/tax-invoice-inquiry`를 실행하면, **교육 완료 후 고객사 담당자에게 세금계산서 발행 진행 가능 여부를 확인하는 메일** 초안을 작성합니다.
사용자가 '검수 완료'라고 입력하면 Gmail 임시보관함에 저장합니다.

---

## 전체 흐름 요약

```
[1단계] 정보 수집 (한 번에)
    ↓
[2단계] 메일 초안 출력
    → 사용자 확인 및 수정 요청 반영
    ↓
[3단계] 사용자가 "검수 완료"라고 입력하면 Gmail 임시보관함에 저장
    → 완료
```

---

## 0단계: 발신자 이름 자동 조회

1. `G:\내 드라이브\Project_managing_tool\.env` 파일을 Read 도구로 읽어 `USER_EMAIL` 값을 파싱합니다.
2. `USER_EMAIL`이 있으면 아래 SQL로 ax-hub에서 display_name을 조회합니다:

```sql
SELECT display_name FROM profiles
WHERE email = 'USER_EMAIL'
LIMIT 1;
```

3. 조회된 `display_name`을 `{발신자이름}`으로 사용합니다.
4. `USER_EMAIL`이 없거나 조회 결과가 없으면 사용자에게 이름을 질문합니다.

---

## 1단계: 정보 수집

인자로 제공되지 않은 항목은 **한 번에 모아서** 질문합니다 (항목별로 나눠 묻지 않음).

수집할 정보:

| 항목 | 필수 여부 | 설명 | 예시 |
|------|----------|------|------|
| 수신자 이름 | 필수 | 고객사 담당자 이름 | 김지혜 |
| 수신자 직함 | 필수 | 담당자 직함 | 팀장 |
| 만족도 점수 | 선택 | 교육 만족도조사 점수 (없으면 해당 문장 생략) | 3.88 |
| 수신자 이메일 | 권장 | Gmail 임시저장 수신자 주소 | contact@samsung.com |

> 발신자 이름은 0단계에서 ax-hub에서 자동 조회한 값을 사용합니다. 다를 경우 사용자가 언급할 것.
> **입력되지 않은 항목은 추가로 묻지 않고 해당 항목 자체를 메일에서 생략합니다.**
> 수신자 이메일을 처음에 받지 못한 경우, '검수 완료' 직전에 요청합니다.

---

## 2단계: 메일 초안 출력

1단계에서 수집한 정보를 바탕으로 메일 초안을 작성합니다.

**목적**: 교육 완료 후 세금계산서 발행 진행 가능 여부 확인

**제목**: `[팀스파르타] 세금계산서 발행 관련 문의드립니다`

**형식**:
```
{수신자이름} {수신자직함}님, 안녕하세요.
팀스파르타 {발신자이름}입니다.

교육과 관련하여 세심하게 챙겨주신 덕분에 교육을 잘 마무리할 수 있었습니다.
[만족도 점수 있을 경우]
교육 전반에 대한 만족도 조사 결과는 {만족도점수}점으로 확인되었습니다.
관련 로우 데이터를 첨부하오니 확인 부탁드립니다.

추가적으로 세금계산서 발행을 진행해도 될지 회신부탁드리겠습니다.
아울러 세금계산서 발행을 위해 사업자등록증 사본도 함께 보내주시면 감사하겠습니다.

감사합니다.
{발신자이름} 드림
```

**조건별 처리**:
- 만족도 점수가 없으면 만족도 문장과 로우 데이터 첨부 안내 두 줄 모두 생략
- 만족도 점수가 있으면 두 줄 모두 포함

메일 초안 출력 후 반드시 아래 문구로 마무리합니다:

> **수정할 내용이 있으면 말씀해 주세요. 내용이 확인되셨으면 "검수 완료"라고 입력해 주시면 Gmail 임시보관함에 저장합니다.**

사용자가 수정을 요청하면 반영 후 다시 출력하고 같은 문구로 대기합니다.

---

## 3단계: Gmail 임시보관함 저장

사용자가 "검수 완료"를 입력하면:

1. 수신자 이메일이 없는 경우 이 시점에 요청합니다.
2. Gmail `create_draft` 도구로 임시보관함에 저장합니다.
3. 저장 완료 후 아래 메시지를 출력합니다:

> **✅ 메일 초안이 Gmail 임시보관함에 저장되었습니다. 추가 수정이 필요하면 말씀해 주세요.**

Gmail 저장 실패 시 오류 내용을 안내하고 재시도 여부를 묻습니다.

---

## Gmail 임시저장 규칙

- 저장은 사용자가 "검수 완료"를 입력한 시점에만 실행합니다. 출력 직후 자동 저장하지 않습니다.
- 수신자 이메일 주소는 1단계에서 수집하며, 미수집 시 "검수 완료" 직전에 요청합니다.
- 저장 후 반드시 "Gmail 임시보관함에 저장되었습니다." 메시지를 출력합니다.
- Gmail 저장 실패 시 오류 내용을 안내하고 재시도 여부를 묻습니다.

```
/tax-invoice-inquiry 김지혜 팀장 3.88

```

---

## 주의사항

- 메일 초안은 사용자 확인 전까지 Gmail에 저장하지 않습니다 — 출력과 동시에 자동 저장 금지
- 수정 요청 시 해당 메일만 재출력하고, 확인 문구로 다시 대기
- **입력되지 않은 항목은 추가로 묻지 않고 해당 항목 전체를 메일에서 생략** (빈 칸이나 "(미정)" 등으로 채우지 않음)
- 파일 저장은 하지 않음 — 출력만 제공
