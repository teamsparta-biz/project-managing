# 교육 결과보고서 생성 스킬

`/결과보고서 [고객사명]` 실행 시, 해당 고객사 폴더의 **설문 응답 엑셀**과 **이전 결과보고서**를 읽어 통계를 집계하고, 팀스파르타 AX교육팀 양식(A4·레드 테마)의 결과보고서를 **MD·HTML·PDF 3종**으로 생성합니다.

## 핵심 원칙

1. **데이터 정직성**: 응답 데이터에서 계산 가능한 수치만 보고에 반영. 없는 데이터(예: 사후 역량 설문)는 만들지 말고 명시하거나 생략.
2. **확정값만 표기**: 강사명·정확 인원·링크 등 미확정 항목은 임시값 대신 행 자체를 생략하거나 `[확인 필요]`로 표기 후 사용자에게 보고.
3. **이전 보고서 구조 준용**: 같은 고객사/유사 과정의 기존 PDF가 있으면 섹션 구조(I~VI)를 따름.
4. **원본 응답 파일 수정 금지**: 엑셀은 읽기만. 산출물만 새로 생성.
5. **임시 스크립트 즉시 삭제**: 집계용 파이썬 스크립트와 덤프 파일은 작업 후 삭제 (CLAUDE.md 규칙).

## 출력 위치

`산출물/결과보고서/[고객사]/` 폴더에 다음 3종 저장:
- `[고객사] 과정명 결과보고서.md` (편집용 원본)
- `[고객사] 과정명 결과보고서.html` (스타일 적용)
- `[고객사] 과정명 결과보고서.pdf` (최종 산출물)

## 실행 흐름

### 1단계: 자료 탐색
- 고객사 폴더(`산출물/결과보고서/[고객사]/`)에서 다음을 찾는다:
  - **만족도 조사 응답** 엑셀 (`*만족도*.xlsx`)
  - **수강생 선정/사전 설문 응답** 엑셀 (`*선정*` 또는 `*사전*.xlsx`)
  - **이전 결과보고서** PDF (구조 참고용)
- 인자로 고객사명이 없으면 폴더 목록을 보여주고 선택받는다.

### 2단계: 환경 준비
프로젝트 `.venv`에 `openpyxl`, `pypdf`가 설치돼 있다. 콘솔 한글 깨짐(cp949)을 피하려고 **결과를 UTF-8 파일로 써서 Read 도구로 읽는다.**

```bash
"g:/내 드라이브/claude_root/.venv/Scripts/python" -m pip install -q openpyxl pypdf  # 미설치 시
```

- **엑셀 파싱**: `openpyxl.load_workbook(f, data_only=True)` → 행을 UTF-8 텍스트로 덤프(`C:\Temp\*.txt`) 후 Read.
- **PDF 텍스트**: `pypdf.PdfReader(pdf).pages[i].extract_text()` → UTF-8 파일로 저장 후 Read. (PDF→이미지 변환은 `pdftoppm` 부재로 불가)

### 3단계: 통계 집계 (만족도 응답)
- **5점 척도 항목**(전반 만족도·강사 전문성·콘텐츠·부가자료 등): 평균, 100점 환산(=평균/5×100)
- **2점 척도(예/아니오)**: 긍정(=2) 응답률 %
- **추천(0~10점) → NPS**: Promoters(9~10)·Passives(7~8)·Detractors(0~6) 분류, `NPS = (P-D)/n×100`, 평균 점수
- **복수 선택(가장 만족/개선점)**: 콤마 분리 후 항목별 빈도 Counter (`없음` 제외)
- 정성 소감(자유 응답): 대표 인용문 3~4개 발췌

### 4단계: 통계 집계 (사전/선정 설문)
- 객관식 항목: 보기별 분포(인원·%)
- 소속/직무: 키워드 빈도
- 자유 응답(자동화 과제 등): 유형별로 묶어 대표 니즈 표로 요약
- **사후 역량 설문이 별도로 있으면** 사전/사후 비교표(향상도 %)로 IV장 구성. 없으면 사전 분석으로 대체하고 그 사실을 명시.

### 5단계: 보고서 작성
표준 섹션 구조:
| 섹션 | 내용 |
|------|------|
| 표지 | 고객사·과정명·기간·담당자 |
| I. 교육과정 개요 | 과정 정보표 + 학습목표 + 세부 커리큘럼 |
| II. 교육 결과물 | 교안·실습자료·발표·(있으면)튜터 피드백 링크 |
| III. 교육 만족도 조사 결과 | 정량(막대그래프)·NPS / 정성(만족·개선 표 + 피드백 카드) |
| IV. 사전 설문 분석 | 수강생 역량·니즈 (또는 사전/사후 역량 변화) |
| V. 과정 회고·성과 요약 | 성과표 + 핵심 인사이트 + 차별점 |
| VI. 종합 결론 | 결론 문단 |

- **MD 먼저 작성** → 같은 내용으로 **HTML** 작성. HTML은 [template.html](template.html)의 CSS·구조를 사용.
- 차트 막대 라벨은 한 줄로 유지(`white-space:nowrap`, 라벨 width 충분히). 두 줄로 깨지면 라벨을 줄이거나 width를 늘린다.
- 후속 교육 제안·슬로건("팀스파르타가 함께…")·작성 주체 문구는 **기본 미포함**. 사용자가 요청할 때만 추가.

### 6단계: PDF 변환 (Chrome 헤드리스)
```bash
CHROME="/c/Program Files/Google/Chrome/Application/chrome.exe"
HTML='G:\...\[고객사] 과정명 결과보고서.html'   # Windows 경로(역슬래시) 사용
PDF='G:\...\[고객사] 과정명 결과보고서.pdf'
"$CHROME" --headless --disable-gpu --no-pdf-header-footer --print-to-pdf="$PDF" "$HTML"
```
- 경로는 반드시 **Windows 형식(역슬래시)** 으로. git-bash `/g/...` 경로는 Chrome가 인식 못함.

### 7단계: 검수 & 정리
- 레이아웃·줄바꿈 확인이 필요하면 HTML을 PNG로 렌더해 Read로 확인:
  ```bash
  "$CHROME" --headless --disable-gpu --hide-scrollbars --force-device-scale-factor=2 \
    --window-size=820,1180 --screenshot="C:\Temp\p1.png" "$HTML"
  ```
  특정 페이지는 임시 복사본에 `body{margin-top:-1123px}`(A4 1장=약 1123px) 오프셋을 넣어 캡처.
- 임시 파이썬 스크립트·덤프·스크린샷 삭제.
- 사용자에게 결과 요약과 함께 `[확인 필요]` 항목(강사·정확 인원·링크 등)을 보고.

## 수정 요청 대응
보고서 완성 후 항목 변경 요청 시: **HTML·MD 모두 수정 → PDF 재생성**. 3종 동기화 유지.

## 자주 하는 실수
| 실수 | 방지 |
|------|------|
| 콘솔에 한글이 깨져 데이터 오독 | 반드시 UTF-8 파일로 덤프 후 Read |
| 없는 데이터(사후 역량 등) 추정 기입 | 없으면 생략·명시. 수치 날조 금지 |
| Chrome PDF 경로 실패 | Windows 역슬래시 경로 사용 |
| 차트 라벨 2줄로 깨짐 | 라벨 width 확대 + nowrap |
| HTML만 고치고 PDF 안 만듦 | 수정 후 항상 PDF 재생성 |
