# 구인 글 슬랙 전송 스킬

교육의 **주강사** 또는 **기술튜터** 구인 글을 작성하여 스파르타 기업교육 슬랙의 해당 풀 채널에 전송합니다.

> **트리거**: 웹앱(교육 진행 현황)의 `주강사 구인` / `기술튜터 구인` 버튼을 누르면 이 스킬을 실행하는 `/recruit-share` 명령이 복사됩니다.

---

## 채널 매핑

| 구분 | 전송 채널 |
|------|-----------|
| 기술튜터 | `#ax교육팀_기술튜터풀` |
| 주강사 | `#ax교육팀_주강사풀` |

명령 인자에 `채널:` 값이 함께 전달되면 그 값을 우선 사용합니다.

---

## 전체 흐름

```
[1단계] 인자 파싱 (구분 / 기업명 / 교육명 / 일정)
    ↓
[2단계] (기술튜터인 경우) 일정별 필요 튜터 인원 수를 사용자에게 질문
    ↓
[3단계] 구인 글 초안 작성 → 미리보기 출력 → 사용자 검수
    ↓
[4단계] 사용자가 "검수완료" 입력 → 해당 채널에 전송
```

---

## 1단계: 인자 파싱

명령에서 아래 정보를 읽습니다. 누락된 항목은 사용자에게 한 번에 모아 질문합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| 구분 | `주강사` 또는 `기술튜터` (필수) | 기술튜터 |
| 채널 | 전송 대상 채널명 (없으면 위 채널 매핑 사용) | ax교육팀_기술튜터풀 |
| 기업명 | 교육 발주 기업 | 삼성인력개발원 |
| 교육명 | 교육 과정명 | '26년 하반기 SVP 신입입문 |
| 일정 | 회차별 날짜·시간·장소 (여러 줄) | 1차: 7월 14일(화) 8:00 ~ 12:00 고양연수원 |

---

## 2단계: (기술튜터만) 일정별 튜터 인원 질문

**구분이 `기술튜터`인 경우에만** 실행합니다. (주강사는 인원 질문 없이 바로 3단계로 진행)

전송 전, 각 일정(회차)별로 필요한 튜터 인원 수를 사용자에게 질문합니다. 한 번에 모아서 묻습니다.

```
각 일정별 필요한 튜터 인원 수를 알려주세요:
1차: 7월 14일(화) 8:00 ~ 12:00 고양연수원 → ?명
2차: 7월 28일(화) 8:00 ~ 12:00 고양연수원 → ?명
...
```

받은 일정별 인원 수가 **모든 회차에서 동일**하면, 각 줄에 반복 표기하지 않고 일정 목록 하단에 "각 회차별 튜터 N인 필요"로 **한 번만** 적습니다.
회차마다 인원 수가 다르면, 가독성 좋게 **각 일정 줄 끝에 `(튜터 N인)` 형식**으로 개별 표기합니다.

동일한 규칙을 **장소**에도 적용합니다: 모든 회차의 장소가 동일하면 각 줄에 반복하지 않고 "교육 일정" 바로 아래에 "장소: {장소}"로 한 번만 적고, 각 회차 줄에서는 장소를 뺍니다. 회차마다 장소가 다르면 각 줄 끝에 장소를 그대로 표기합니다.

---

## 3단계: 구인 글 초안 작성 및 미리보기

아래 양식으로 구인 글을 작성합니다.

**기술튜터 양식 (회차별 인원이 다른 경우):**

```
@channel [{기업명}] {교육명} 교육에 참여 가능한 튜터님들의 많은 지원 부탁드립니다!

교육 일정
1차: 7월 14일(화) 8:00 ~ 12:00 고양연수원 (튜터 4인)
2차: 7월 28일(화) 8:00 ~ 12:00 고양연수원 (튜터 2인)
...

교육 내용은 아래 이미지 참고
```

**기술튜터 양식 (회차별 인원·장소가 동일한 경우 — 한 번만 표기):**

```
@channel [{기업명}] {교육명} 교육에 참여 가능한 튜터님들의 많은 지원 부탁드립니다!

교육 일정
장소: {장소}
1차: 7월 14일(화) 8:00 ~ 12:00
2차: 7월 28일(화) 8:00 ~ 12:00
...

각 회차별 튜터 {N}인 필요

교육 내용은 아래 이미지 참고
```

**주강사 양식:** (일정별 인원 표기 없음, 장소는 위와 동일한 규칙 적용)

```
@channel [{기업명}] {교육명} 교육에 참여 가능한 강사님들의 많은 지원 부탁드립니다!

교육 일정
장소: {장소}
1차: 7월 14일(화) 8:00 ~ 12:00
2차: 7월 28일(화) 8:00 ~ 12:00
...

교육 내용은 아래 이미지 참고
```

작성한 초안을 대화창에 미리보기로 출력하고, 전송 대상 채널을 함께 표시한 뒤 아래 문구로 마무리합니다:

> **전송 대상: #{채널}**
> **수정할 내용이 있으면 말씀해 주세요. 확인되셨으면 "검수완료"라고 입력해 주시면 슬랙 채널에 전송합니다.**

수정 요청이 있으면 반영 후 다시 미리보기를 출력하고 같은 문구로 대기합니다.

> **중요**: 사용자가 "검수완료"라고 입력하기 전까지 절대 전송하지 않습니다.

---

## 4단계: 슬랙 채널 전송

사용자가 "검수완료"를 입력하면:

1. `slack_search_channels`로 대상 채널(`ax교육팀_기술튜터풀` 또는 `ax교육팀_주강사풀`)을 조회하여 채널 ID를 확보합니다.
2. `slack_send_message`로 해당 채널에 구인 글 원문 그대로 전송합니다. (요약·수정 없이 미리보기와 동일하게)
3. 전송 완료 후 아래 메시지를 출력합니다:

> **✅ #{채널} 채널에 구인 글을 전송했습니다.**

전송 실패 시 오류 내용을 안내하고 재시도 여부를 묻습니다.

---

## 사용 예시

```
/recruit-share
구분: 기술튜터
채널: ax교육팀_기술튜터풀
기업명: 삼성인력개발원
교육명: '26년 하반기 SVP 신입입문
일정:
1차: 7월 14일(화) 8:00 ~ 12:00 고양연수원
2차: 7월 28일(화) 8:00 ~ 12:00 고양연수원
→ 일정별 튜터 인원 질문 → 초안 미리보기 → "검수완료" → 채널 전송
```

---

## 주의사항

- 사용자가 "검수완료"를 입력한 시점에만 전송합니다. 미리보기 직후 자동 전송 금지.
- 기술튜터 구인은 일정별 인원 수를 받기 전까지 미리보기를 확정하지 않습니다.
- 회차별 필요 튜터 인원이 전부 동일하면 각 줄에 반복 표기하지 말고 일정 하단에 한 번만 적습니다. 회차마다 다를 때만 줄별로 표기합니다.
- 회차별 장소가 전부 동일하면 각 줄에 반복 표기하지 말고 "교육 일정" 바로 아래에 "장소: {장소}"로 한 번만 적습니다. 회차마다 장소가 다를 때만 줄별로 표기합니다.
- 주강사 구인은 일정별 인원 표기를 하지 않습니다.
- `@channel` 멘션은 양식 그대로 포함하여 전송합니다.
- 입력되지 않은 항목(교육명 등)은 빈 칸으로 두지 말고 사용자에게 확인합니다.
