# 튜터 온보딩 스킬

튜터가 확정되면 슬랙 단톡방을 만들고 교육 안내 메시지를 전송하는 워크플로우입니다.

> **트리거 명령어**: `slack_dm` 또는 `/tutor-onboarding 기업명 교육명:교육명` 을 입력하면 이 스킬이 실행됩니다.

> **핵심 원칙**: 강사·기술튜터·일정·장소·교안 링크는 사용자에게 입력받거나 웹앱 명령 인자에서 가져오지 않고, **ax-hub DB에서 직접 조회**합니다. 웹앱 `SlackDM방 개설` 버튼은 기업명·교육명만 전달합니다.

---

## 전체 흐름 요약

```
[1단계] 인자 파싱 (기업명 / 교육명) → ax-hub course 조회
    ↓
[2단계] 회차별 일정 + 강사·기술튜터 배정 조회 (ax-hub)
    ↓
[3단계] 장소·교안 링크 조회 (ax-hub)
    ↓
[4단계] 슬랙 사용자 ID 확보 (DB slack_id 우선, 없으면 검색)
    ↓
[5단계] 그룹 DM(단톡방) 생성
    ↓
[6단계] 메시지 초안(표 형식) 검토 요청 → 사용자 확인
    ↓
[7단계] 사용자가 "검수완료" 입력 → 전송
```

---

## 1단계: 인자 파싱 및 course 조회

명령 인자에서 `기업명`(첫 번째 위치 인자)과 `교육명:`을 추출합니다.

```sql
SELECT c.id AS course_id, c.title AS training_name, c.place
FROM courses c
JOIN deals d ON c.deal_id = d.id
JOIN clients cl ON d.client_id = cl.id
WHERE cl.name ILIKE '%{기업명}%'
ORDER BY c.lecture_start DESC NULLS LAST
LIMIT 5;
```

- 결과가 1건이면 해당 `course_id` 사용.
- 여러 건이면 `교육명`(부분 일치)으로 좁히고, 그래도 여러 건이면 사용자에게 어떤 교육인지 물어봅니다.
- `기업명`이 없거나 course를 찾지 못하면, 기존 방식대로 사용자에게 강사·튜터·일정·장소·교안 링크를 직접 질문해 진행합니다 (ax-hub 조회 불가 시 폴백).

---

## 2단계: 회차별 일정 + 강사·기술튜터 배정 조회

```sql
SELECT
  cs.date,
  i.name,
  i.slack_id,
  qc.category::text AS role_category
FROM course_rounds cr
JOIN course_sessions cs ON cs.round_id = cr.id
LEFT JOIN assignments a ON a.course_session_id = cs.id
LEFT JOIN qualification_catalog qc ON qc.id = a.qualification_id
LEFT JOIN instructors i ON i.id = a.instructor_id
WHERE cr.course_id = '{course_id}'
ORDER BY cs.date, qc.category, i.name;
```

- 날짜(회차)별로 그룹핑합니다. 회차 번호는 날짜 오름차순으로 부여합니다 (1회차, 2회차, ...).
- 역할 분류: `main_instructor` → 강사, `tech_tutor`/`mentor` → 기술튜터.
- 특정 회차에 `main_instructor` 배정이 없으면 그 회차 강사는 **"미정"**으로 표시합니다 (임의로 다른 회차 강사를 채우지 않습니다).
- 전체 강사 목록(중복 제거, "미정" 제외)과 전체 기술튜터 목록(중복 제거)을 별도로 모아둡니다 — 그룹 DM 구성원 및 본문 상단 명단에 사용합니다.

---

## 3단계: 장소·교안 링크 조회

```sql
SELECT c.place, w.shorten_url, w.full_url
FROM courses c
LEFT JOIN workbooks w ON w.course_id = c.id
WHERE c.id = '{course_id}';
```

- 장소는 `c.place`를 사용합니다.
- 교안 링크는 `shorten_url`을 우선 사용하고 없으면 `full_url`을 사용합니다.
- 둘 다 없으면 해당 항목을 사용자에게 질문합니다 (없는 상태로 메시지에 빈 칸을 남기지 않습니다).

---

## 4단계: 슬랙 사용자 ID 확보

2단계에서 모은 전체 강사·기술튜터 각각에 대해:

- `instructors.slack_id`가 DB에 있으면 그대로 사용합니다.
- 없으면 `slack_search_users`로 이름 또는 이메일(`notion_email`/`email`)로 검색합니다.
- 검색으로도 찾지 못하면 그룹 DM 및 발송 대상에서 제외하고, 최종 결과 안내 시 누구를 제외했는지 알려줍니다.

---

## 5단계: 그룹 DM(단톡방) 생성

`slack_create_conversation` 툴을 사용합니다.

```
user_ids: [강사1 user_id, 강사2 user_id (있는 경우), 튜터1 user_id, 튜터2 user_id, ...]
```

> 주의: 그룹 DM은 채널이 아닌 DM 형태로 생성합니다. user_ids에 본인(봇)은 제외합니다. 최대 8명까지 가능하므로 초과 시 사용자에게 안내합니다.

생성 후 반환된 `channel_id`를 저장합니다.

---

## 6단계: 메시지 초안 검토 요청

메시지를 전송하기 전, 사용자에게 초안을 보여주고 검수를 요청합니다.

**회차가 2개 이상인 경우** — 표 형식(Slack 마크다운 표)으로 회차별 배정을 보여줍니다:

```
강사·튜터님들 안녕하세요, {기업명} {교육명} 교육 안내드립니다!

* 강사: {전체 강사 목록, 쉼표 구분} 강사님 ({강사 미정 회차가 있으면 "N회차 강사는 미정" 추가})
* 기술튜터: {전체 기술튜터 목록, 쉼표 구분} 님

* 교육 일정 및 회차별 배정

| 회차 | 일정 | 강사 | 기술튜터 |
|---|---|---|---|
| 1회차 | {날짜(요일) 시간} | {강사 또는 "미정"} | {기술튜터, 쉼표 구분} |
| ... | ... | ... | ... |

* 교육 장소 : {장소}
* 교안 : {교안 링크}

* 강사님께서는 이전에 전달 받으신 커리큘럼으로 교안 개발이 완료되신 다면 말씀 한 번 부탁드리겠습니다.
감사합니다!
```

**회차가 1개인 경우** — 표 없이 단일 일정으로 안내합니다:

```
강사·튜터님들 안녕하세요, {기업명} {교육명} 교육 안내드립니다!

* 교육 일정 : {날짜(요일) 시간}
* 교육 장소 : {장소}
* 교안 : {교안 링크}

* 강사: {강사명 또는 "미정"} 강사님
* 기술튜터: {기술튜터 목록, 쉼표 구분} 님

* 강사님께서는 이전에 전달 받으신 커리큘럼으로 교안 개발이 완료되신 다면 말씀 한 번 부탁드리겠습니다.
감사합니다!
```

날짜 표기는 `YY/MM/DD(요일)` 형식을 사용합니다 (예: `26/07/20(월)`).

미리보기 출력 후 아래 문구로 마무리합니다:

```
구성원: {강사 목록}(강사), {기술튜터 목록}(기술튜터)
단톡방 ID: {channel_id}

검수가 완료되셨으면 "검수완료"라고 입력해 주세요.
```

> **중요**: "차주 진행되는" 같은 상대적 시점 표현은 사용하지 않습니다. 사용자가 "검수완료"를 입력하기 전까지 절대 메시지를 전송하지 않습니다. 수정 요청이 있으면 반영 후 다시 미리보기를 보여줍니다.

---

## 7단계: 교육 안내 메시지 전송

사용자가 "검수완료"라고 입력하면 `slack_send_message` 툴로 생성된 단톡방에 메시지를 전송합니다.

---

## 오류 대응

| 상황 | 대응 |
|------|------|
| ax-hub에서 course를 찾지 못함 | 사용자에게 기업명·교육명을 다시 확인하거나, 강사·튜터·일정·장소·교안 링크를 직접 질문 |
| 슬랙 사용자를 찾을 수 없음 | 발송 대상에서 제외하고 사용자에게 안내 |
| 그룹 DM 생성 실패 | 이미 동일 구성원 DM이 있을 수 있음 — 기존 DM 채널 ID를 찾아 메시지 전송 |

---

## 완료 후 사용자에게 전달할 내용

- 생성된 단톡방 링크 (slack 메시지 링크)
- 전송된 메시지 미리보기
- 구성원 명단 확인
- 슬랙 ID 미확인으로 제외된 인원이 있으면 안내
