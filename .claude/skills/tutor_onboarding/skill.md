# 튜터 온보딩 스킬
 
튜터가 확정되면 슬랙 단톡방을 만들고 교육 안내 메시지를 전송하는 워크플로우입니다.
 
> **트리거 명령어**: `slack_dm` 을 입력하면 이 스킬이 즉시 실행됩니다.
 
## 필요한 정보 수집
 
스킬 시작 전, 아래 정보가 모두 있는지 확인하세요. 없는 항목은 사용자에게 질문하세요.
 
### 단톡방 구성원
- **강사**: 슬랙 이름 또는 이메일 (1명 이상, 여러 명일 수 있음)
- **튜터들**: 슬랙 이름 또는 이메일 (1명 이상)
### 교육 안내 메시지용 정보
- **기업명**: (예: "조선일보") — 단톡방 주제 설정에 사용
- **교육명**: (예: "통일과나눔 교육")
- **강사 수**: 1명 또는 2명
  - 강사가 2명인 경우 → 회차별 일정 따로 입력
    - **1회차 날짜 및 시간**: (예: "26/05/21 (목) 13:30~17:30")
    - **2회차 날짜 및 시간**: (예: "26/05/28 (목) 13:30~17:30")
  - 강사가 1명인 경우 → 단일 일정 입력
    - **날짜 및 시간**: (예: "26/05/21 (목) 13:30~17:30")
- **주소**: (예: "서울 중구 세종대로21길 52 조선일보광화문빌딩 8층")
- **교안 링크**: (예: "https://b2bsparta.com/0513-502") — 반드시 포함

## 실행 순서
 
### Step 1: 슬랙 사용자 ID 조회
 
강사(1명 이상)와 튜터 각각의 슬랙 사용자 ID를 조회합니다.
 
- `slack_search_users` 툴을 사용하여 이름 또는 이메일로 검색
- 스파르타 기업교육 워크스페이스 기준으로 검색
- 강사가 여러 명인 경우 모두 조회하여 user_id를 메모해 둡니다
### Step 2: 그룹 DM(단톡방) 생성
 
`slack_create_conversation` 툴을 사용합니다.
 
```
is_private: true
user_ids: [강사1 user_id, 강사2 user_id (있는 경우), 튜터1 user_id, 튜터2 user_id, ...]
```
 
> 주의: 그룹 DM은 채널이 아닌 DM 형태로 생성합니다. user_ids에 본인(봇)은 제외합니다.
 
생성 후 반환된 `channel.id`를 저장합니다.
 
### Step 3: 단톡방 이름 설정
 
`slack_create_conversation` 으로 생성된 그룹 DM의 대화 이름을 설정합니다. 슬랙 API의 `conversations.rename` 에 해당하는 툴을 사용합니다.
 
- **이름 형식**: `{기업명}_{교육명}` (예: `조선일보_통일과나눔교육`)
- 슬랙 채널명 규칙상 공백 없이, 소문자·숫자·하이픈·언더스코어만 허용됨 — 한글은 그대로 사용 가능하나 공백은 언더스코어(`_`)로 대체
- channel_id는 Step 2에서 저장한 값 사용
- 실패 시: 그룹 DM은 이름 변경이 제한될 수 있음 — 사용자에게 수동 설정 안내

### Step 4: 단톡방 주제 설정
 
슬랙 API의 `conversations.setTopic` 에 해당하는 툴을 사용합니다.
 
- **주제 형식**: `{기업명}_{교육명}` (예: `조선일보_통일과나눔 교육`)
- channel_id는 Step 2에서 저장한 값 사용
### Step 5: 메시지 초안 검토 요청

메시지를 전송하기 전, 사용자에게 초안을 보여주고 검수를 요청합니다.

아래 형식으로 대화창에 출력합니다:

```
📋 전송 예정 메시지 미리보기

[작성된 메시지 전체]

구성원: 강사명, 튜터명1, 튜터명2
단톡방 ID: {channel_id}

검수가 완료되셨으면 "검수완료"라고 입력해 주세요.
```

> **중요**: 사용자가 "검수완료"라고 입력하기 전까지 절대 메시지를 전송하지 않습니다. 수정 요청이 있으면 메시지를 수정한 뒤 다시 미리보기를 보여줍니다.

### Step 6: 교육 안내 메시지 전송

사용자가 "검수완료"라고 입력하면 `slack_send_message` 툴로 생성된 단톡방에 메시지를 전송합니다.
 
**메시지 양식 — 강사 1명 (단일 회차):**
 
```
튜터님들 안녕하세요, 차주 진행되는 {기업명} {교육명} 교육 안내드립니다!
 
* 교육 일정 : {날짜 및 시간}
* 교육 장소 : {주소}
* 교안 : {교안 링크}
   * 강사님께서는 이전에 전달 받으신 커리큘럼으로 교안 개발이 완료되신 다면 말씀 한 번 부탁드리겠습니다.
감사합니다!
```
 
**메시지 양식 — 강사 2명 (2회차):**
 
```
튜터님들 안녕하세요, 차주 진행되는 {기업명} {교육명} 교육 안내드립니다!
 
* 1회차 교육 일정 : {1회차 날짜 및 시간} ({1회차 강사명} 강사님)
* 2회차 교육 일정 : {2회차 날짜 및 시간} ({2회차 강사명} 강사님)
* 교육 장소 : {주소}
* 교안 : {교안 링크}
   * {1회차 강사명} 강사님께서는 이전에 전달 받으신 커리큘럼으로 교안 개발이 완료되신 다면 말씀 한 번 부탁드리겠습니다.
감사합니다!
```

## 오류 대응
 
| 상황 | 대응 |
|------|------|
| 사용자를 찾을 수 없음 | 사용자에게 정확한 이름/이메일 재확인 요청 |
| 그룹 DM 생성 실패 | 이미 동일 구성원 DM이 있을 수 있음 — 기존 DM 채널 ID를 찾아 메시지 전송 |
| 주제 설정 실패 | 그룹 DM은 주제 설정이 제한될 수 있음 — 사용자에게 수동 설정 안내 |

## 완료 후 사용자에게 전달할 내용
 
- 생성된 단톡방 링크 (slack 채널 URL)
- 설정된 주제 (`기업명_교육명`)
- 전송된 메시지 미리보기
- 구성원 명단 확인
