# 교육 메일 초안 생성 스킬

사용자가 `/email-draft`를 실행하면, **교육 시작 전 고객사 담당자에게 인사 및 교육 개요를 안내하는 첫 발송 메일** 초안을 작성합니다.
사용자가 '검수 완료'라고 입력하면 Gmail 임시보관함에 저장합니다.

---

## 첫 발송 메일 (교육 안내)

### 전체 흐름 요약

```
[1단계] 정보 수집 (한 번에)
    ↓
[2단계] 첫 발송 메일 초안 출력
    → 사용자 확인 및 수정 요청 반영
    ↓
[3단계] 사용자가 "검수 완료"라고 입력하면 Gmail 임시보관함에 저장
    → 완료
```

---

### 1단계: 정보 수집

인자로 제공되지 않은 항목은 **한 번에 모아서** 질문합니다 (항목별로 나눠 묻지 않음).

수집할 정보:

| 항목 | 필수 여부 | 설명 | 예시 |
|------|----------|------|------|
| 고객사명 | 권장 | 교육을 받는 기업 (ax-hub 조회에 사용) | 삼성전자 |
| 교육 주제 | 권장 | 교육 제목/과목 (ax-hub 조회에 사용) | 생성형 AI 활용 실무 |
| 수신자 이메일 | 권장 | Gmail 임시저장 수신자 주소 | contact@samsung.com |
| 교육 일정 | 선택 | 날짜 + 요일 + 시간 (없으면 생략) | 2026년 04월 30일 목요일 13:00~17:00 |
| 교육 장소 | 선택 | 건물명 (없으면 생략) | 삼성전자 수원사업장 |
| 장소 상세 | 선택 | 층/호수 등 | 본관 7층 세미나실 |
| 강사명 + 연락처 | 선택 | 없으면 생략 | 김연준 / 010-7353-7724 |
| 기술튜터 | 선택 | 이름 + 연락처 (없으면 생략) | 김가은 / 010-7176-1627 |
| 교안 링크 | 선택 | Notion 또는 공유 URL (없으면 생략) | https://b2bsparta.com/0415-463 |
| 교안 현황 | 선택 | 없으면 생략 | 강사와 작업 중, 내일 최종 회신 예정 |

> 발신자 정보는 기본값 **팀스파르타 송찬호 매니저** 사용. 다를 경우 사용자가 언급할 것.
> **입력되지 않은 항목은 추가로 묻지 않고 해당 항목 자체를 메일에서 생략합니다.**
> 수신자 이메일을 처음에 받지 못한 경우, '검수 완료' 직전에 요청합니다.

---

### 2단계: 첫 발송 메일 초안 출력

1단계에서 수집한 정보를 바탕으로 메일 초안을 작성합니다.

**목적**: 교육 진행 전 처음으로 고객사 담당자에게 인사 및 교육 개요 안내

**제목**: `[팀스파르타] {고객사명} {교육주제} 교육 안내드립니다`

**형식**:
```
안녕하세요
처음 인사드립니다. 팀스파르타 {발신자이름} {직함} 입니다.

이번 {고객사명} 임직원 대상 「{교육주제}」 교육과 관련하여 연락 드립니다.
교육 세부 사항을 아래와 같이 공유 드리며, 검토 부탁 드립니다.

교육 일정
- {일정}

교육 장소
- {장소}
- {상세장소 또는 "구체적인 장소를 안내해 주시면 감사하겠습니다."}

교육 주제
- {교육주제}

교육 진행 인원
- 강사 : {강사명} / {연락처}
- 기술튜터 1 : {튜터명} / {연락처}
- 기술튜터 2 : {튜터명} / {연락처}
- 강사 외에 기술 멘토링 도움을 위해 튜터 2인도 교육 당일 참여 예정입니다.  ← 튜터 참여 시 항상 포함

교안
[교안 링크 입력 시 아래 전체 포함, 미입력 시 교안현황만 표시, 둘 다 없으면 섹션 전체 생략]
- 링크 : {교안링크}
- 교안은 노션 링크에서 확인 가능하며, 수업 또한 노션을 통해 진행될 예정입니다.
- 첨부드린 링크에서 각 목차의 페이지를 열어보면 관련 내용 확인이 가능합니다.
- 노션에 익숙하지 않은 분들을 위해, 교육 시작 전 강사님께서 교안 활용 방안에 대해 설명해주실 예정입니다.
- 노션 접속이 어려운 경우 PDF로 추출하여 전달 또한 가능하오니 편히 말씀 부탁드리겠습니다.
[교안 링크 없이 교안 현황만 있을 경우]
- {교안현황}

교육 준비와 관련하여 필요하신 사항이 있으시면 편하게 연락 주시기 바랍니다.

감사합니다.
{발신자이름} 드림
```

메일 초안 출력 후 반드시 아래 문구로 마무리합니다:

> **수정할 내용이 있으면 말씀해 주세요. 내용이 확인되셨으면 "검수 완료"라고 입력해 주시면 Gmail 임시보관함에 저장합니다.**

사용자가 수정을 요청하면 반영 후 다시 출력하고 같은 문구로 대기합니다.

---

### 3단계: Gmail 임시보관함 저장

사용자가 "검수 완료"를 입력하면:

1. 수신자 이메일이 없는 경우 이 시점에 요청합니다.
2. Gmail `create_draft` 도구로 임시보관함에 저장합니다.
3. 저장 완료 후 아래 메시지를 출력합니다:

> **✅ 메일 초안이 Gmail 임시보관함에 저장되었습니다. 추가 수정이 필요하면 말씀해 주세요.**

Gmail 저장 실패 시 오류 내용을 안내하고 재시도 여부를 묻습니다.

---

## Gmail 임시저장 규칙

- 저장은 사용자가 "검수 완료"를 입력한 시점에만 실행합니다. 출력 직후 자동 저장하지 않습니다.
- 수신자 이메일 주소는 1단계에서 수집하며, 미수집 시 "검수 완료" 직전에 요청합니다.
- 저장 후 반드시 "Gmail 임시보관함에 저장되었습니다." 메시지를 출력합니다.
- Gmail 저장 실패 시 오류 내용을 안내하고 재시도 여부를 묻습니다.

---

## 사용 예시

```
/email-draft
→ 정보 수집 → 메일 초안 출력 → "검수 완료" → Gmail 임시보관함 저장
```

```
/email-draft 삼성전자 "생성형 AI 실무" "04월 30일 목요일 13:00~17:00"
→ 인자 활용, 나머지 정보 질문 후 동일 단계 진행
```

---

## 주의사항

- 메일 초안은 사용자 확인 전까지 Gmail에 저장하지 않습니다 — 출력과 동시에 자동 저장 금지
- 수정 요청 시 해당 메일만 재출력하고, 확인 문구로 다시 대기
- **입력되지 않은 항목은 추가로 묻지 않고 해당 항목 전체를 메일에서 생략** (빈 칸이나 "(미정)" 등으로 채우지 않음)
- 상세 장소 미입력 시 "구체적인 장소 안내 요청" 문구 자동 포함 (장소 자체가 없으면 장소 섹션 전체 생략)
- 파일 저장은 하지 않음 — 출력만 제공
