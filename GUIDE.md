## 1. 시작하기

웹앱을 처음 열면 **이름 선택 팝업**이 뜹니다.

1. 등록된 이름 목록에서 본인 이름을 클릭합니다.
2. 이름이 없으면 **이름 추가** 버튼을 눌러 이름을 등록한 뒤 선택합니다.
3. **시작하기** 버튼을 클릭하면 해당 이름에 저장된 데이터가 불러와집니다.

> 이름별로 데이터가 분리 저장됩니다. 우측 상단의 이름 배지를 클릭하면 언제든지 사용자를 전환할 수 있습니다.

## 2. 사전 설정 — Gmail · Slack · 환경변수

스킬(이메일 초안, Slack DM방 개설 등)을 사용하려면 **Claude Code에 Gmail과 Slack을 연결**하고 **`.env` 파일을 설정**해야 합니다. 최초 1회만 진행하면 됩니다.

---

### 2-1. Gmail 연결

Gmail 연결은 **claude.ai 웹사이트의 통합(Integrations) 설정**에서 진행합니다. 연결 후 Claude Code가 Gmail 임시보관함에 메일 초안을 자동 저장할 수 있습니다.

**연결 방법**:

1. 브라우저에서 [claude.ai](https://claude.ai) 에 로그인합니다.
2. 좌측 하단 또는 프로필 메뉴에서 **Settings(설정)** 으로 이동합니다.
3. **Integrations** 탭을 클릭합니다.
4. 목록에서 **Gmail** 을 찾아 **Connect(연결)** 버튼을 클릭합니다.
5. Google 계정 인증 팝업이 열리면 업무용 Google 계정(`@teamsparta.co`)으로 로그인합니다.
6. Gmail 접근 권한 요청 화면에서 **허용**을 클릭합니다.
7. claude.ai 통합 목록에서 Gmail 항목에 **연결됨(Connected)** 표시가 뜨면 완료입니다.

**연결 확인**:

Claude Code 터미널에서 아래를 입력해 오류 없이 응답이 오면 정상 연결된 것입니다.

```
Gmail 임시보관함에 테스트 메일 초안 만들어줘
```

> Gmail 연결은 claude.ai 계정에 묶여 있습니다. Claude Code 데스크탑 앱을 같은 계정으로 로그인하면 자동으로 사용 가능합니다.

---

### 2-2. Slack 연결

Slack 연결도 동일하게 **claude.ai 통합 설정**에서 진행합니다. 연결 후 Claude Code가 Slack 채널 검색, 사용자 조회, 메시지 전송 등을 수행할 수 있습니다.

**연결 방법**:

1. 브라우저에서 [claude.ai](https://claude.ai) 에 로그인합니다.
2. **Settings → Integrations** 탭으로 이동합니다.
3. 목록에서 **Slack** 을 찾아 **Connect(연결)** 버튼을 클릭합니다.
4. Slack 인증 팝업이 열리면 업무용 Slack 워크스페이스(팀스파르타)를 선택합니다.
5. 권한 요청 화면에서 **허용(Allow)** 을 클릭합니다.
6. claude.ai 통합 목록에서 Slack 항목에 **연결됨(Connected)** 표시가 뜨면 완료입니다.

**연결 확인**:

Claude Code 터미널에서 아래를 입력해 채널 목록이 반환되면 정상 연결된 것입니다.

```
Slack에서 채널 목록 검색해줘
```

> Slack 연결 시 **워크스페이스 전체에 대한 접근 권한**이 부여됩니다. 메시지 전송 전 Claude Code가 항상 확인을 요청하므로, 실수로 메시지가 전송될 걱정은 하지 않아도 됩니다.

---

### 2-3. `.env` 파일 설정

프로젝트 루트(`G:\내 드라이브\Project_managing_tool\`)에 `.env` 파일을 생성합니다. `.env.example` 파일을 복사해서 사용하세요.

**`.env` 파일 내용**:

```env
USER_EMAIL=자신의이메일@teamsparta.co
```

`USER_EMAIL` 하나만 설정하면 됩니다. `email-draft`, `tax-invoice-inquiry` 스킬이 이 이메일로 ax-hub에서 **발신자 이름을 자동 조회**합니다. 설정하지 않으면 스킬 실행 시 매번 이름을 직접 입력해야 합니다.

**설정 방법**:

```
# .env.example 복사
copy .env.example .env

# 텍스트 편집기로 .env 열기 후 본인 이메일로 수정
USER_EMAIL=본인이메일@teamsparta.co
```

### 2-4. 연결 상태 요약

| 서비스 | 연결 방법 | 용도 |
|--------|----------|------|
| **Gmail** | claude.ai → Settings → Integrations → Gmail 연결 | 이메일 초안 임시보관함 저장 |
| **Slack** | claude.ai → Settings → Integrations → Slack 연결 | SlackDM방 개설, 교육운영 공유 |
| **ax-hub** | 별도 설정 불필요 (팀 MCP 서버에 자동 연결) | 교육 데이터 조회 및 동기화 |
| **`.env`** | 파일 직접 편집 | 발신자 이름 자동 조회 |

## 3. 화면 구성 개요

```
┌─────────────────────────────────────────────────────────────┐
│ 교육 프로젝트 진행 현황          [교육 가져오기]  [사용자]  │
├─────────────────────────────────────────────────────────────┤
│ [기업명 입력...] [기업 추가] [업무 추가] [테마] [초기화]   │
├─────────────────────────────────────────────────────────────┤
│ [진행 중 N] [교육 완료 N]                                   │
├───────────────┬─────────────────────────────────────────────┤
│               │  ◀ 교육 전 ▶  │       ◀ 교육 후 ▶         │
│  기업 정보    ├───┬───┬───┬───┼───┬───────────────────────┤
│               │   │   │   │   │   │                       │
├───────────────┼───┴───┴───┴───┴───┴───────────────────────┤
│  진행률 / 액션 버튼    │  회차별 일정                       │
└─────────────────────────────────────────────────────────────┘
│                     업무 특이사항                            │
└─────────────────────────────────────────────────────────────┘
```

- **헤더 행**: 업무 목록이 "교육 전" / "교육 후" 섹션으로 구분됩니다.
- **기업 행**: 왼쪽에 기업 정보, 오른쪽 열마다 각 업무의 진행 상태가 표시됩니다.
- **푸터 행**: 기업별 진행률과 Claude Code 연동 버튼이 있습니다.
- **하단**: 자유롭게 메모할 수 있는 업무 특이사항 영역이 있습니다.

## 4. 기업(프로젝트) 관리

### 기업 추가

1. 툴바의 **기업명 입력** 필드에 기업명을 입력합니다.
2. **Enter** 키 또는 **기업 추가** 버튼을 클릭합니다.
3. 새 기업 행이 "진행 중" 탭에 추가됩니다.

### 기업 정보 입력

각 기업 행의 왼쪽 셀에서 다음 정보를 편집할 수 있습니다:

| 필드 | 설명 |
|------|------|
| 기업명 | 상단 굵은 글씨 입력 필드 |
| 교육명 | 북마크 아이콘 옆 입력 필드 |
| 장소 | 핀 아이콘 옆 입력 필드 |
| 담당자 이메일 | 메일 아이콘 옆 입력 필드 (이메일 초안·세금계산서 버튼에 자동 반영) |
| 교안 링크 | 교안 URL이 있으면 책 아이콘과 함께 링크로 표시됨 |
| 교육 상태 | 클릭 시 **미설정 → 세팅중 → 교육중 → 교육종료** 순으로 순환 |
| 메모 | 최하단 노트 아이콘 옆 자유 텍스트 |

### 교육 완료 처리 / 복원

- **교육 완료** 버튼: 해당 기업을 "교육 완료" 탭으로 이동합니다.
- **복원** 버튼 (교육 완료 탭): 다시 "진행 중" 탭으로 옮깁니다.

### 기업 삭제

- **삭제** 버튼을 클릭하면 확인 팝업 후 영구 삭제됩니다. 되돌릴 수 없습니다.

## 5. 업무 관리

### 업무 추가

1. 툴바의 **업무 추가** 버튼 또는 테이블 헤더 오른쪽 끝의 **＋** 버튼을 클릭합니다.
2. **업무명**과 **구분** (교육 전 / 교육 후)을 입력하고 **추가** 버튼을 클릭합니다.

### 업무 편집

- 업무 헤더를 **더블클릭**하거나, 헤더 하단의 **연필 아이콘**을 클릭합니다.
- 업무명과 구분(교육 전/후)을 수정한 뒤 **저장**을 클릭합니다.

### 업무 삭제

- 헤더 하단의 **X 아이콘**을 클릭하면 확인 팝업 후 삭제됩니다.
- 삭제된 업무의 모든 기업 진행 상태도 함께 제거됩니다.

### 업무 순서 변경

- 업무 헤더를 **좌우로 드래그**하여 순서를 바꿀 수 있습니다.
- 파란 구분선이 삽입 위치를 표시합니다.

> 기본 제공 업무 목록 (교육 전): 담당자 인계 이메일 송부, 주강사 구인, 기술튜터 구인, 교안 확인, 교안 피드백 진행, 교안 자료 이관 완료, 고객사 교안 송부, 강사·튜터 리마인더
> 기본 제공 업무 목록 (교육 후): 만족도 조사 진행

## 6. 진행 상태 관리

각 기업 × 업무 셀을 **클릭**하면 상태가 순환됩니다.

| 상태 | 표시 | 배경색 |
|------|------|--------|
| 미시작 | ○ | 흰색 |
| 진행 중 | … | 주황색 |
| 완료 | ✓ | 초록색 |

- 각 기업 행 하단에 **진행률(%)** 과 완료/전체 수가 표시됩니다.
- 진행 중 업무는 0.5점으로 계산하여 부분 진행률에 반영됩니다.

## 7. 회차별 일정 관리

각 기업의 푸터 영역(진행률 오른쪽)에 **회차별 일정** 패널이 있습니다.

### 일정 입력

- **날짜**: 날짜 선택기로 교육 날짜를 지정합니다.
- **시간**: 시작 시간 / 종료 시간을 입력합니다.
- **강사명** / **튜터명**: 각 회차별로 담당자를 입력합니다.

### 회차 추가 / 삭제

- **회차 추가** 버튼으로 새 회차를 추가합니다.
- 회차가 2개 이상일 때 회차 헤더의 **X 버튼**으로 삭제합니다.
- 화살표 버튼으로 회차 간 이동합니다.

### 일정 상태 배지

날짜·시간이 입력되면 **예정 / 진행 중 / 종료됨** 배지가 자동으로 표시됩니다.

## 8. 업무 특이사항 메모

화면 하단의 **업무 특이사항** 영역에 자유롭게 메모할 수 있습니다.

### 탭 구성

| 탭 | 설명 |
|----|------|
| 특이사항 | 자유 메모 편집 (Markdown 지원) |
| 업무 완료 | 정리된 완료 항목 보관 |
| 미리보기 | 작성한 Markdown을 렌더링해서 확인 |

### 편집 단축키

| 단축키 | 동작 |
|--------|------|
| `Tab` | 들여쓰기 (공백 2칸) |
| `Shift+Tab` | 내어쓰기 |
| `Enter` (리스트 줄) | 다음 리스트 항목 자동 생성 |
| `Ctrl+Z` | 되돌리기 |
| `Ctrl+Y` / `Ctrl+Shift+Z` | 다시 실행 |
| `Ctrl+Shift+X` | 선택 영역에 취소선(~~텍스트~~) 적용 |

### 정리하기 기능

- **정리하기** 버튼을 클릭하면 취소선(`~~텍스트~~`)으로 표시된 항목이 **업무 완료** 탭으로 자동 이동합니다.
- 원래 섹션(헤딩) 기준으로 그룹핑하여 날짜 도장(예: `[2026.06.28 정리]`)과 함께 저장됩니다.

## 9. 탭 전환 (진행 중 / 교육 완료)

테이블 상단의 탭으로 전환합니다.

- **진행 중**: `archived: false`인 기업만 표시됩니다. 날짜 오름차순 정렬.
- **교육 완료**: `archived: true`인 기업만 표시됩니다. 완료된 교육 이력 보관용.

## 10. 테마 설정

툴바 오른쪽의 테마 토글로 전환합니다.

## 11. 스킬 사용법

이 웹앱은 **Claude Code** 와 연동하여 반복 업무를 자동화합니다. 아래 스킬은 Claude Code 터미널에서 슬래시 명령으로 실행합니다.

웹앱의 각 버튼을 클릭하면 해당 스킬 명령어가 **클립보드에 자동 복사**됩니다. 복사된 명령어를 Claude Code에 붙여넣어 실행하세요.

### 스킬 설치 및 업데이트 방법

스킬은 이 저장소의 `.claude/skills/` 폴더에 파일로 관리됩니다. **별도 설치 과정은 없으며, 저장소를 클론하면 스킬이 함께 포함됩니다.**

**처음 사용할 때 (저장소 클론)**:

```bash
git clone <저장소 URL>
cd Project_managing_tool
```

클론 후 Claude Code를 이 폴더에서 열면 `.claude/skills/` 안의 스킬이 자동으로 인식됩니다.

**스킬이 업데이트되었을 때**:

스킬 내용이 수정되거나 새 스킬이 추가된 경우, `git pull`로 내려받아야 반영됩니다.

```bash
git pull
```

### sync-ax-hub — 교육 데이터 자동 동기화

**목적**: ax-hub DB에서 담당자 기준으로 교육 목록을 조회하여 웹앱에 자동으로 반영합니다.

**실행 방법**:

1. 웹앱 상단의 **교육 가져오기** 버튼을 클릭합니다.
2. `/sync-ax-hub 담당자:이름` 형식의 명령어가 클립보드에 복사됩니다.
3. Claude Code에 붙여넣어 실행합니다.

```
/sync-ax-hub 담당자:홍길동
```

**동작 순서**:

1. ax-hub `profiles` 테이블에서 담당자 이름으로 이메일 조회
2. 담당자의 교육 목록 조회 (기업명, 교육명, 장소, 상태)
3. 회차별 강사·튜터 정보 조회
4. 회차별 교육 날짜·시간 조회
5. 교안 링크 조회
6. 기존 `state.json` 과 병합 (수동 입력 데이터 보존)
7. 로컬 서버 또는 파일에 저장
8. 동기화 결과 요약 출력

**course_status 매핑**:

| ax-hub 상태 | 웹앱 표시 |
|------------|----------|
| `setup` | 세팅중 |
| `operation` | 교육중 |
| `closed` | 교육 완료 탭으로 이동 |
| `stopped` | 교육 완료 탭으로 이동 |

> 동기화 후 브라우저에서 **F5**를 눌러 새로고침하면 반영된 내용을 확인할 수 있습니다.

### email-draft — 교육 안내 이메일 초안

**목적**: 교육 시작 전 고객사 담당자에게 발송하는 첫 인사 및 교육 안내 이메일 초안을 작성하고 Gmail 임시보관함에 저장합니다.

**실행 방법**:

1. 기업 행의 **이메일 초안** 버튼을 클릭합니다.
2. `/email-draft 기업명 / 교육명: ... / 교육일자: ...` 형식의 명령어가 복사됩니다.
3. Claude Code에 붙여넣어 실행합니다.

```
/email-draft 삼성전자 / 교육명: 생성형 AI 실무
/email-draft 삼성전자 "생성형 AI 실무" "04월 30일 목요일 13:00~17:00"
```

**진행 단계**:

```
[1단계] 정보 수집 (교육 일정, 장소, 강사·튜터명, 교안 링크 등)
    ↓
[2단계] 이메일 초안 출력 및 수정 반영
    ↓
[3단계] "검수 완료" 입력 시 Gmail 임시보관함에 저장
```

**수집 정보 목록**:

| 항목 | 필수 | 설명 |
|------|------|------|
| 고객사명 | 권장 | 교육 받는 기업명 |
| 교육 주제 | 권장 | 교육 제목 |
| 수신자 이메일 | 권장 | 담당자 이메일 |
| 교육 일정 | 선택 | 날짜 + 시간 |
| 교육 장소 | 선택 | 건물명 |
| 강사명 + 연락처 | 선택 | |
| 기술튜터 + 연락처 | 선택 | |
| 교안 링크 | 선택 | Notion URL 등 |

> 입력하지 않은 항목은 메일에서 자동으로 생략됩니다 (빈 칸으로 채우지 않음).
> 발신자 이름은 `.env`의 `USER_EMAIL` 기준으로 ax-hub에서 자동 조회합니다.

### tax-invoice-inquiry — 세금계산서 발행 확인 메일

**목적**: 교육 완료 후 고객사 담당자에게 세금계산서 발행 진행 가능 여부를 확인하는 메일 초안을 작성합니다.

**실행 방법**:

1. 기업 행의 **세금계산서발행** 버튼을 클릭합니다.
2. `/tax-invoice-inquiry 교육명: ... / 담당자이메일: ...` 형식의 명령어가 복사됩니다.
3. Claude Code에 붙여넣어 실행합니다.

```
/tax-invoice-inquiry
/tax-invoice-inquiry 김지혜 팀장 3.88
```

**진행 단계**:

```
[1단계] 수신자 이름, 직함, 만족도 점수 수집
    ↓
[2단계] 이메일 초안 출력 및 수정 반영
    ↓
[3단계] "검수 완료" 입력 시 Gmail 임시보관함에 저장
```

**수집 정보 목록**:

| 항목 | 필수 | 설명 |
|------|------|------|
| 수신자 이름 | 필수 | 고객사 담당자 이름 |
| 수신자 직함 | 필수 | 예: 팀장, 담당자 |
| 만족도 점수 | 선택 | 없으면 해당 문장 생략 |
| 수신자 이메일 | 권장 | Gmail 임시저장 수신자 주소 |

### notion-share — 노션 페이지 공유

**목적**: 노션 페이지에 특정 사람을 초대하거나 권한을 설정해야 할 때 사용합니다.

**실행 방법**:

Claude Code에서 직접 실행합니다.

```
/notion-share
노션 공유해줘
노션 페이지에 초대해줘
```

**진행 단계**:

```
[1단계] 노션 페이지 URL 확인 → 페이지 제목 검증
    ↓
[2단계] 이메일로 워크스페이스 사용자 조회 (멤버 여부 확인)
    ↓
[3단계] 공유 준비 결과 출력 + 노션에서 직접 공유하는 방법 안내
```

**권한 수준**:

| 권한 | 설명 |
|------|------|
| 읽기 전용 | 내용만 볼 수 있음 |
| 댓글 가능 | 읽기 + 댓글 작성 |
| 편집 가능 | 내용 수정 가능 (기본값) |
| 전체 권한 | 공유 설정까지 변경 가능 |

> 워크스페이스 외부 사용자의 경우 게스트 초대 방식으로 안내됩니다.
